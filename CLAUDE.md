# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Chrome extension (Manifest V3) that replaces hard-to-read Japanese fonts on websites with **Noto Sans JP** (for general text) and **UDEV Gothic JPDOC** (for monospace/code). Written in vanilla JavaScript/CSS with no runtime framework. No test suite, no linter, no CI/CD pipeline.

## Build Commands

```bash
# Full build (icons + CSS + screenshots)
npm run build

# Individual generators
npm run generate-css          # Regenerate css/replacefont-extension.css from font configs
npm run generate-icons        # Generate PNG icons from SVG (requires sharp)
npm run generate-screenshots  # Generate Web Store screenshots (requires puppeteer)
npm run convert-fonts         # Convert TTF to WOFF2 (requires ttf2woff2)

# Package for Chrome Web Store (version sync + build + ZIP)
# Windows:
powershell -File zip.ps1
# Unix/macOS:
bash zip.sh
```

Dependencies (`sharp`, `puppeteer`, `ttf2woff2`) are in `optionalDependencies` — ZIP packaging works without `npm install`. Run `npm install` only when you need to regenerate icons/screenshots/fonts.

## Generated Files (do not edit manually)

- `css/replacefont-extension.css` — Generated by `scripts/generate-css.js`
- `icons/icon-*.png` — Generated from `icons/icon.svg` by `scripts/generate-icons.js`
- `webstore/images/*.png` — Generated from `webstore/screenshots/*.html` by `scripts/generate-screenshots.js`

## Architecture

### Extension Entry Points

- **`preload-fonts.js`** — Main content script (~370 lines). Runs at `document_start` on all frames. Handles CSS injection, font preloading, and Shadow DOM detection via MutationObserver + TreeWalker chunked scanning.
- **`inject.js`** — Injected into MAIN world (page context, not content script sandbox). Patches `Element.prototype.attachShadow` to mark hosts with `data-rfs-shadow` attribute, then dispatches `replace-font-shadow-created` event so the content script can detect closed Shadow DOMs.
- **`popup/`** — Display-only popup (no user interaction). Shows extension info and version (fetched from manifest at runtime via `popup.js`). Glassmorphism CSS design.

### CSS Generation Pipeline

`scripts/generate-css.js` is the source of truth for which fonts get replaced. It defines:
- `GOTHIC_FONT_FAMILIES` (~60 entries) — Sans-serif/serif fonts → Noto Sans JP
- `MONO_FONT_FAMILIES` (~20 entries) — Monospace fonts → UDEV Gothic JPDOC
- Hiragino weight variants (W1-W9) are auto-generated
- CSS uses `__REPLACE_FONT_BASE__` placeholder, replaced at runtime by the content script with the actual extension URL

### Font Replacement Strategy

1. **`@font-face` redefinition** — Each target font family gets a `@font-face` rule that points to local Noto Sans JP / UDEV Gothic JPDOC or the bundled WOFF2 fallback
2. **CSS variable overrides** — Common CSS custom properties (`--font-sans`, `--font-mono`, `--tw-font-sans`, etc.) are overridden with `!important`
3. **Selector-based overrides** — `pre`, `code`, `kbd`, `samp`, and code-related class selectors get explicit monospace font assignment

### Shadow DOM Handling (two-pronged approach)

1. **Open Shadow DOM** — Content script uses MutationObserver + TreeWalker to find elements with `.shadowRoot` and injects CSS via Constructable Stylesheets (`adoptedStyleSheets`)
2. **Closed Shadow DOM** — `inject.js` patches `attachShadow` in page context, marks host with `data-rfs-shadow`, dispatches event back to the content script

### Key Implementation Details

- **CSS injection:** Prefers Constructable Stylesheets (`CSSStyleSheet.replaceSync()`), falls back to `<style>` tags for unsupported roots, with retry logic (max 3 retries)
- **Caching:** CSS text in `fixedCSSCache` (Map), CSSStyleSheet objects in `sheetCache` (Map)
- **DOM scanning:** Chunked processing (200 elements per chunk via `requestIdleCallback`) to avoid UI freezes
- **Font preloading:** Both `<link rel="preload">` tags and CSS Font Loading API (`FontFace.load()`)
- **Duplicate prevention:** `_replaceFontApplied` / `_replaceFontInProgress` flags on shadow roots

## Version Management

Version is managed in `package.json` as the single source of truth. The `zip.sh`/`zip.ps1` scripts auto-sync it to `manifest.json`, `README.md`, `popup/popup.html`, `docs/index.html`, and webstore screenshot HTML files via regex replacement.

## Adding a New Font to Replace

1. Add the font family name to `GOTHIC_FONT_FAMILIES` or `MONO_FONT_FAMILIES` in `scripts/generate-css.js`
2. Run `npm run generate-css` to regenerate the CSS
3. If a new CSS variable is commonly used for that font, add it to the `variableOverrides` section in `generate-css.js`

## Important Constraints

- **Never use universal selectors (`* {}`)** in any CSS — they break icon fonts (Font Awesome, Material Icons, etc.) by overriding their `font-family`. Use explicit element selectors instead.
- **`preload-fonts.js` is a hot path** — it runs on every single page load at `document_start`. Avoid unnecessary DOM queries, and batch operations where possible (e.g., Shadow DOM events use `queueMicrotask` batching).
- **Shadow DOM event batching** — When `attachShadow` is called repeatedly (YouTube, Disney+, etc.), the `replace-font-shadow-created` event handler deduplicates via `queueMicrotask` to avoid O(n²) `querySelectorAll` calls.

## Testing

No automated test suite. Manual testing via HTML files in `debug/` directory (shadow DOM detection, font loading, iframe handling). See `debug/TEST_GUIDE.md` for manual test procedures. Load the extension unpacked in Chrome and open the debug HTML files to validate behavior.
